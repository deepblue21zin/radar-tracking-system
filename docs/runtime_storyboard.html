Capstone / Real-Time mmWave Tracking

실시간 객체추적 시스템을 “설명 가능하고 시연 가능한 상태”까지 끌어올리는 프로젝트

1. 이 프로젝트의 최종목표 및 무엇을 하려는가

    4. 현재 구현된 것

    이미 구현되어 있고 지금 프로젝트의 강점으로 말할 수 있는 항목들이다.

    실시간 런타임 파이프라인

    TLV parse -> world transform -> filter -> DBSCAN -> tracker -> viewer/report/control 흐름이 하나의 runtime 경로로 통합되어 있다.

    설치 보정

    sensor_yaw_deg, sensor_pitch_deg, sensor_height_m 기준으로 sensor frame을 world frame으로 보정한다.

    시각화 구조 개선

    viewer는 read/process와 draw를 분리해 최신 snapshot만 그리도록 구성되어 있다.

    로그/리포트 자동화

    run 종료 시 frame CSV, overview PNG, experiment report, performance_log, Doxygen portal이 자동 갱신된다.

    5. 앞으로 구현해야 할 것 REQ

    지금부터는 있으면 좋은 기능이 아니라, 시연 안정성과 연구 확장을 위해 실제로 필요한 요구사항으로 정리했다.

    REQ-1

    사람 1명 = track 1개에 가깝게

    split track merge, ground-range gate, association 개선으로 single-person continuity를 높여야 한다.

    REQ-2

    대각선 오인식 제거

    pose calibration wizard 또는 로그 기반 yaw/pitch 추천 도구가 필요하다.

    REQ-3

    시연 모드 안정화

    viewer FPS pressure를 더 줄이고 replay/demo mode를 제공해야 한다.

    REQ-4

    parser 관측성 향상

    resync 원인 분류, invalid packet 상세 카운터, UART 상태 요약이 더 필요하다.

    REQ-5

    평가 자동화

    front/back, lateral walk, idle scene을 preset 실험으로 자동 돌리고 비교할 수 있어야 한다.

    REQ-6

    control end-to-end 검증

    실제 belt output까지 포함한 latency / safety 검증이 남아 있다.

    7. 어떻게 해결할 수 있을지

    문제를 한 번에 다 고치기보다, 원인에 맞는 파일과 파라미터부터 건드리는 순서가 중요하다.

    대각선 이동처럼 보임

    world-coordinate 보정 이후에도 걷기 축이 틀어지면 calibration UI 또는 로그 기반 yaw/pitch 추천 도구를 추가한다.

    Best touch points: runtime_pipeline.py / runtime_params.json

    한 사람을 여러 track으로 봄

    adaptive eps + post-merge + primary/secondary 표시 전략으로 split ratio를 낮춘다.

    Best touch points: dbscan_cluster.py / kalman_tracker.py

    zero-track 긴 구간

    ground-range 기반 gate, report_miss_tolerance 조정, far-field filtering 재설계가 필요하다.

    Best touch points: noise_filter.py / kalman_tracker.py

    시연 딜레이

    draw pressure를 더 낮추고 latest-only queue 정책을 유지하며 replay mode를 별도 제공한다.

    Best touch points: live_rail_viewer.py

    8. 파라미터 값이 달라지면 결과값이 어떻게 달라지는가: 연계성 시뮬레이터

    9. 논문으로 발전시킬 수 있는 아이디어

    지금 구조는 단순 캡스톤 결과물로 끝내기보다, calibration / filtering / split reduction / runtime observability 쪽으로 연구 확장이 가능하다.

    자가 보정 기반 설치 캘리브레이션

    사람의 반복 보행 궤적만으로 yaw/pitch/height 오차를 추정하는 방법을 연구 주제로 확장할 수 있다.

    world-coordinate aware filtering

    sensor range가 아니라 ground-range, corridor distance, belt axis distance 기준 filtering을 정리할 수 있다.

    split-track reduction for sparse mmWave

    temporal merge + velocity consistency로 사람 1명의 분할을 줄이는 방법을 논문화할 수 있다.

    runtime observability framework

    로그/리포트/HTML portal 자동화를 실시간 mmWave 시스템의 observability framework로 정리할 수 있다.
