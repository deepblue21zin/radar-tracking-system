  - 기존에는 runtime loop callback에서 draw까지 같이 수행해서, Matplotlib stall이 serial read와 frame 처리 지연으로 이어질 수 있었다.
  - 시연에서는 중간 표시 프레임을 모두 그리는 것보다, 최신 상태를 덜 늦게 보여주는 편이 더 중요하다.

### 3. 실험 저널과 자동 run 리포트 구조 추가
- 수정 내용:
  - `docs/experiment_journal.md`를 추가해 지금까지의 문제 확인 -> 가설 -> 수정 -> 검증 -> 다음 액션 흐름을 정리했다.
  - `src/reporting/runtime_experiment_report.py`를 추가해 run 종료 시 `docs/experiment_reports/YYYY-MM-DD/<run_id>.md` 형식의 실험 리포트를 자동 생성하고, 인덱스 `docs/experiment_reports/README.md`도 자동 갱신하도록 했다.
  - `src/reporting/performance_log_report.py`를 추가해 `docs/performance_log.md`도 run 종료 시 자동 갱신되도록 바꿨다.
- `tlv_parse_runner.py`에 `--experiment-title`, `--experiment-problem`, `--experiment-hypothesis`, `--experiment-change`, `--experiment-next-step` 인자를 추가했다.
- 수정 이유:
  - 기존 `performance_log.md`만으로는 수치 요약은 되지만, 왜 이런 가설을 세우고 무엇을 바꿨는지가 한눈에 보이지 않았다.
  - 앞으로는 run이 끝날 때 바로 구조화 리포트가 남아, 취업용 포트폴리오나 실험 회고에 바로 쓸 수 있게 하는 게 목표다.
