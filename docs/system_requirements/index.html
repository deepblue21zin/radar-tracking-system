REQ-INTERFACE-CONTRACT

실무에서는 알고리즘보다 입출력 계약과 안전한 실패 동작이 더 중요할 때가 많습니다. 이 섹션은 radar input, tracking output, health status, control packet 계약을 정리합니다.

Current Strength12-byte fixed packet + CRC8
Main GapEMPTY_SCENE vs SENSOR_UNHEALTHY 분리 부족
Current Data Contractframe / cluster / track / control output 설명 존재
Next Stephealth bitfield와 tracking state 확장
ContractRequired FieldsCurrent State
RadarInputFramemagic word, packet bytes, frame number, num obj, TLV count, x/y/z/v/range/snr/noisePartial
TrackingOutputtrack_id, x/y/z, vx/vy, confidence, state, age_frames, missed_countPartial
HealthStatusEMPTY_SCENE_VALID, SENSOR_UNHEALTHY, STALE_DATA, PARSER_DEGRADED, DROPPED_FRAME_HIGH, OVERRUNPlanned
ControlPacketheader, sequence, command, speed ratio, event, flags, zone distance, closing speed, crc8Implemented / needs versioning

가장 중요한 정책 분리

  • EMPTY_SCENE_VALID: 사람이 없는 정상 장면
  • SENSOR_UNHEALTHY: 레이더를 신뢰할 수 없는 상태
  • 이 둘은 같은 의미가 아니며, 동시에 true가 되면 안 됨

제어 패킷 확장 추천

  • packet version 추가
  • health flags bitfield 추가
  • timeout policy 명시
  • reserved bits 확보

REQ-SAFETY-POLICY

stale data, malformed frame, overrun, dropped burst에서 어떻게 fail-safe로 가는지를 명확히 정의한 정책 섹션입니다. 핵심은 unhealthy 상황에서 RESUME을 금지하는 health gate입니다.

Already Existstimeout stop, emergency stop, parser counters
Main Riskunhealthy와 empty scene이 혼동될 수 있음
Target Policyhealth bad면 RESUME 금지
Verificationfault injection + timeout stop 검증
ConditionDetectionHost ActionSTM32 ActionResume Allowed?
EMPTY_SCENE_VALIDparser healthy + frame age 정상 + detection 0RESUME 허용normal heartbeatYes
STALE_DATA`frame_age_ms > 100`ALARM 또는 송신 중단timeout stopNo
MALFORMED_FRAME_BURSTparse fail / resync burstALARM, health degradestop or hold slowNo
OVERRUNbuffer/queue overrundegraded mode, latest-onlyhold safe commandPrefer No
DROPPED_FRAME_HIGHrecent dropped rate threshold 초과SLOW 또는 ALARMslow or stopPrefer No

추천 Health Gate

  • `frame_age_ms > 100`
  • `parse_failures_delta > 0` in short window
  • `resync_rate` threshold 초과
  • `dropped_frame_burst` threshold 초과
  • `buffer_overrun_count > 0` recent window

Recovery Rule 예시

  • stale recovery: healthy frame 3개 연속 후 clear
  • malformed burst recovery: cooldown window 후 clear
  • dropped recovery: recent window 정상화 후 clear
  • overrun recovery: utilization 안정화 후 clear
