REQ-IF

REQ-INTERFACE-CONTRACT

레이더 입력, 내부 추적 결과, health 상태, STM32 제어 패킷의 계약을 정리한 문서입니다. 핵심은 EMPTY_SCENE와 SENSOR_UNHEALTHY를 같은 의미로 다루지 않는 것입니다.

Critical Policy Split

EMPTY_SCENE_VALID

parser healthy + 최근 frame 정상 + detection 0개. 사람이 없는 정상 장면을 뜻합니다.

SENSOR_UNHEALTHY

parser 또는 transport health가 깨져 레이더를 신뢰할 수 없는 상태를 뜻합니다.

Why It Matters

이 둘을 분리하지 않으면 unhealthy 상황에서도 RESUME으로 해석될 수 있어 안전 정책이 약해집니다.

Control Packet Extension

  • packet version 추가
  • health flags bitfield 추가
  • timeout policy 명시
  • reserved bits 확보
