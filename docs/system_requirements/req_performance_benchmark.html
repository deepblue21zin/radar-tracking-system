Required Benchmark Axes

CategoryMetricWhy
Throughput`avg_fps`지속 처리 성능
Latency`avg/max/p95 parser latency`packet decode 안정성
Latency`avg/max/p95 pipeline latency`end-to-end responsiveness
Continuity`parse_failures`, `resync_events`, `dropped_frames_estimate`실시간 신뢰성
Utilization`cpu_percent`, `peak_rss_mb`host 부하 근거
Embedded Fit`post_init_alloc_count`, `heap_peak_bytes`양산형 설계 적합성

Recommended Code Changes

Summary 확장

`max_parser_latency_ms`, `max_pipeline_latency_ms`, `p95_parser_latency_ms`, `p95_pipeline_latency_ms` 추가

Host 부하 계측

`cpu_percent_avg`, `peak_rss_mb` 수집

Embedded fit

startup 이후 dynamic allocation 0회 목표 및 allocator hook

Portfolio Form

면접/포트폴리오에서는 "avg parser latency 2.8 ms -> 1.9 ms, p95 pipeline latency 41 ms -> 19 ms, dropped frame estimate 23 -> 4"처럼 before/after로 표현하는 것이 좋습니다.
